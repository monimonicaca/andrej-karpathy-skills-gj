# Karpathy Guidelines

Use this skill to keep coding changes minimal, explicit, and verifiable.

## Core rules

- Think before coding: surface assumptions and clarify ambiguity.
- Simplicity first: do only what was asked.
- Surgical changes: touch only related code and clean up your own unused artifacts.
- Goal-driven execution: define success criteria and verify them.
